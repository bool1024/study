XML解析之Digester组件使用详解
1.Digester简介  
JAVA中解析XML常用的两个API：SAX和DOM。

SAX和DOM定义了不同的分析和处理XML的方法，SAX是基于事件的API，而DOM是基于树的API。对于基于事件处理的分析器而言，分析事件通过回调方法直接报告给应用程序。使有DOM，你需要将XML文档转换成内存中的树结构，应用程序再遍历树来分析这个文档。

Digester是基于SAX API的，因此Digester也是基于事件处理的，其内部定义了许多事件处理规则，某些匹配模式被发现时，就会触发对应的事件处理规则。使用Digester解析XML文档，我们只需告诉要做什么，至于如何解析XML文档我们不用管，这就是Digester优于SAX和DOM的地方。

由于Digester使用方便性的优势，Digester组件被广泛使用，典型的使用场景是Tomcat框架，Tomcat中解析server.xml和web.xml时都是采用Digester方式实现的。

3.Digester使用详解  
Digester匹配模式可以通过两种方式提供：通过Java代码直接显示设置、通过配置文件形式提供。基于匹配模式的两种提供方式，Digester解析xml有两种实现方式。下面将详细介绍这两种Digester解析xml的实现方式（下面代码依赖commons-digester3-3.2.jar、commons-beanutils-1.9.3.jar、commons-collections-3.2.0.jar）。

（2）通过Java代码提供匹配模式
（a）提供解析代码，如下所示： 
  1. public AllPersons readPersons(File file) throws IOException, SAXException {
  2.     // Digester句柄
  3.     Digester digester = new Digester();
  4.     // 定义目标对象，并压入Digester栈
  5.     AllPersons persons = new AllPersons();
  6.     digester.push(persons);
  7.     // 创建Person对象
  8.     digester.addObjectCreate("persons/person", Person.class);
  9.     // 填充Person对象的"age"属性和"name"属性
  10.     digester.addSetProperties("persons/person", "name", "name");
  11.     digester.addBeanPropertySetter("persons/person/age", "age");
  12.     // 创建Address对象，并填充属性值，然后将Address对象作为属性填充给Person对象
  13.     digester.addObjectCreate("persons/person/address", Address.class);
  14.     digester.addBeanPropertySetter("persons/person/address/street", "street");
  15.     digester.addBeanPropertySetter("persons/person/address/city", "city");
  16.     System.out.println(Address.class.getName());
  17.     digester.addSetNext("persons/person/address", "setAddress", Address.class.getName());
  18.     // 创建CreditCard对象，并填充属性值，然后将CreditCard对象作为属性填充给Person对象
  19.     digester.addObjectCreate("persons/person/creditcard", CreditCard.class);
  20.     // 通过方法调用设置属性值
  21.     digester.addCallMethod("persons/person/creditcard", "addParams", 2, new Class[]{int.class, String.class});
  22.     digester.addCallParam("persons/person/creditcard/limit", 0);
  23.     digester.addCallParam("persons/person/creditcard/bank", 1);
  24.     // 将xml中的attribute映射成JavaBean的属性值
  25.     digester.addSetProperties("persons/person/creditcard/limit");
  26.     digester.addSetProperties("persons/person/creditcard/limit", "pwd", "password");
  27.     digester.addSetNext("persons/person/creditcard", "addCreditCard", CreditCard.class.getName());
  28.     // 将Person对象填充给返回的目标对象
  29.     digester.addSetNext("persons/person", "addPerson", Person.class.getName());
  30.     return (AllPersons) digester.parse(file);
  31. }

（3）通过配置文件提供匹配模式
（a）提供匹配模式文件rule_set.xml，内容如下： 
  1. <?xml version="1.0" encoding="GB2312"?>
  2. <digester-rules>
  3.     <pattern value="persons">
  4.         <set-properties-rule />
  5.         <pattern value="person">
  6.             <object-create-rule classname="com.huawei.zbs.xml.digester.vo.Person" />
  7.             <set-properties-rule />          
  8.             <pattern value="age">
  9.                 <call-method-rule methodname="setAge" paramcount="0"  paramtypes="java.lang.Integer" />
  10.             </pattern>
  11.             <pattern value="address">
  12.                 <object-create-rule classname="com.huawei.zbs.xml.digester.vo.Address" />
  13.                 <set-properties-rule />              
  14.                 <pattern value="street">
  15.                     <call-method-rule methodname="setStreet" paramcount="0" />
  16.                 </pattern>
  17.                 <pattern value="city">
  18.                     <bean-property-setter-rule />
  19.                 </pattern>
  20.                 <set-next-rule methodname="setAddress" />
  21.             </pattern>
  22.             <pattern value="creditcard">
  23.                 <object-create-rule classname="com.huawei.zbs.xml.digester.vo.CreditCard" />
  24.                 <set-properties-rule />
  25.                 <call-method-rule methodname="addParams" paramcount="2" paramtypes="java.lang.Integer,java.lang.String"/>
  26.                 <call-param-rule pattern="limit" paramnumber="0"/>
  27.                 <call-param-rule pattern="bank" paramnumber="1" />                              
  28.                 <pattern value="limit">
  29.                     <set-properties-rule>
  30.                         <alias attr-name="code" prop-name="code"/>
  31.                         <alias attr-name="pwd" prop-name="password"/>
  32.                     </set-properties-rule>
  33.                 </pattern>                 
  34.                 <set-next-rule methodname="addCreditCard" />
  35.             </pattern>
  36.             <set-next-rule methodname="addPerson" />
  37.         </pattern>
  38.     </pattern>
  39. </digester-rules>
