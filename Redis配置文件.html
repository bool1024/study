在Redis目录下有一个redis.conf文件，这个文件即是Redis的配置文件。我们可以在启动的时候用以下命令来读取配置文件启动。 
[root@localhost ~]# ./redis-server /path/redis.conf

Redis的一些度量单位
Redis配置中对单位的大小写不敏感，1GB、1Gb、和1gB都是相同的。由此说明，Redis只支持bytes，不支持bit单位 
# 1k => 1000 bytes
# 1kb => 1024 bytes
# 1m => 1000000 bytes
# 1mb => 1024*1024 bytes
# 1g => 1000000000 bytes
# 1gb => 1024*1024*1024 bytes

redis.conf 配置项说明如下： 
  1. # Redis默认不是以守护进程的方式运行，可以通过该配置项修改，使用yes启用守护进程
  2. daemonize no
  4. # 当Redis以守护进程方式运行时，Redis默认会把pid写入/var/run/redis.pid文件，可以通过pidfile指定
  5. pidfile /var/run/redis.pid
  7. # 指定Redis监听端口，默认端口为6379，作者在自己的一篇博文中解释了为什么选用6379作为默认端口，因为6379在手机按键上MERZ对应的号码，而MERZ取自意大利歌女Alessia Merz的名字
  8. port 6379
  10. # 绑定的主机地址
  11. bind 127.0.0.1
  13. # 当 客户端闲置多长时间后关闭连接，如果指定为0，表示关闭该功能
  14. timeout 300
  16. # 指定日志记录级别，Redis总共支持四个级别：debug、verbose、notice、warning，默认为verbose
  17. loglevel verbose
  19. # 日志记录方式，默认为标准输出，如果配置Redis为守护进程方式运行，而这里又配置为日志记录方式为标准输出，则日志将会发送给/dev/null
  20. logfile stdout
  22. # 设置数据库的数量，默认数据库为0，可以使用SELECT <dbid>命令在连接上指定数据库id
  23. databases 16
  25. # 指定在多长时间内，有多少次更新操作，就将数据同步到数据文件，可以多个条件配合
  26. # save <seconds> <changes>
  27. # Redis默认配置文件中提供了三个条件，分别表示900秒（15分钟）内有1个更改，300秒（5分钟）内有10个更改以及60秒内有10000个更改：
  28. save 900 1
  29. save 300 10
  30. save 60 10000

  32. # 指定存储至本地数据库时是否压缩数据，默认为yes，Redis采用LZF压缩，如果为了节省CPU时间，可以关闭该选项，但会导致数据库文件变的巨大
  33. rdbcompression yes
  35. # 指定本地数据库文件名，默认值为dump.rdb
  36. dbfilename dump.rdb
  38. # 指定本地数据库存放目录
  39. dir ./
  41. # 设置当本机为slav服务时，设置master服务的IP地址及端口，在Redis启动时，它会自动从master进行数据同步
  42. slaveof <masterip> <masterport>
  44. # 当master服务设置了密码保护时，slav服务连接master的密码
  45. masterauth <master-password>
  47. # 设置Redis连接密码，如果配置了连接密码，客户端在连接Redis时需要通过AUTH <password>命令提供密码，默认关闭
  48. requirepass foobared
  50. # 设置同一时间最大客户端连接数，默认无限制，Redis可以同时打开的客户端连接数为Redis进程可以打开的最大文件描述符数，如果设置 maxclients 0，表示不作限制。当客户端连接数到达限制时，Redis会关闭新的
  51. # 连接并向客户端返回max number of clients reached错误信息
  52. maxclients 128
  54. # 指定Redis最大内存限制，Redis在启动时会把数据加载到内存中，达到最大内存后，Redis会先尝试清除已到期或即将到期的Key，当此方法处理 后，仍然到达最大内存设置，将无法再进行写入操作，但仍然可以进行读取
  55. # 操作。Redis新的vm机制，会把Key存放内存，Value会存放在swap区
  56. maxmemory <bytes>
  58. # 指定是否在每次更新操作后进行日志记录，Redis在默认情况下是异步的把数据写入磁盘，如果不开启，可能会在断电时导致一段时间内的数据丢失。因为 redis本身同步数据文件是按上面save条件来同步的，所以有的数据
  59. # 会在一段时间内只存在于内存中。默认为no
  60. appendonly no
  62. # 指定更新日志文件名，默认为appendonly.aof
  63. appendfilename appendonly.aof
  65. # 指定更新日志条件，共有3个可选值：
  66. # no：表示等操作系统进行数据缓存同步到磁盘（快）
  67. # always：表示每次更新操作后手动调用fsync()将数据写到磁盘（慢，安全）
  68. # everysec：表示每秒同步一次（折衷，默认值）
  69.   appendfsync everysec
  71. # 指定是否启用虚拟内存机制，默认值为no，简单的介绍一下，VM机制将数据分页存放，由Redis将访问量较少的页即冷数据swap到磁盘上，访问多的页面由磁盘自动换出到内存中（在后面的文章我会仔细分析Redis的VM机制）
  72. vm-enabled no
  74. # 虚拟内存文件路径，默认值为/tmp/redis.swap，不可多个Redis实例共享
  75. vm-swap-file /tmp/redis.swap
  77. # 将所有大于vm-max-memory的数据存入虚拟内存,无论vm-max-memory设置多小,所有索引数据都是内存存储的(Redis的索引数据 就是keys),也就是说,当vm-max-memory设置为0的时候,其实是所有value都存在于
  78. # 磁盘。默认值为0
  79. vm-max-memory 0
  81. # Redis swap文件分成了很多的page，一个对象可以保存在多个page上面，但一个page上不能被多个对象共享，vm-page-size是要根据存储的 数据大小来设定的，作者建议如果存储很多小对象，page大小最好设置为
  82. # 32或者64bytes；如果存储很大大对象，则可以使用更大的page，如果不 确定，就使用默认值
  83. vm-page-size 32
  85. # 设置swap文件中的page数量，由于页表（一种表示页面空闲或使用的bitmap）是在放在内存中的，，在磁盘上每8个pages将消耗1byte的内存。
  86. vm-pages 134217728
  88. # 设置访问swap文件的线程数,最好不要超过机器的核数,如果设置为0,那么所有对swap文件的操作都是串行的，可能会造成比较长时间的延迟。默认值为4
  89. vm-max-threads 4
  91. # 设置在向客户端应答时，是否把较小的包合并为一个包发送，默认为开启
  92. glueoutputbuf yes
  94. # 指定在超过一定的数量或者最大的元素超过某一临界值时，采用一种特殊的哈希算法
  95. hash-max-zipmap-entries 64
  96. hash-max-zipmap-value 512

  98. # 指定是否激活重置哈希，默认为开启（后面在介绍Redis的哈希算法时具体介绍）
  99. activerehashing yes
  101. # 指定包含其它的配置文件，可以在同一主机上多个Redis实例之间使用同一份配置文件，而同时各个实例又拥有自己的特定配置文件
  102. include /path/to/local.conf
