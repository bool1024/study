常见问题：
1.maven工程中spring读取applicationContext.xml时需要将applicationContext.xml文件放入src/main/resources中
2.lazy-init:false表示spring启动时立刻实例化（lazy-init设置只对scop属性为singleton的bean起作用）
3.不要在spring的配置里配置上XSD的版本号，因为如果没有配置版本号，取得就是当前jar里的XSD文件，减少了各种风险。而且这样约定大于配置的方式很优雅。
4.spring启动时执行函数的配置:（启动时执行com.huawei.u2020.nbiinventoryservice.job.JobManager中的init函数） 
<bean id="initjob" class="com.huawei.u2020.nbiinventoryservice.job.JobManager" init-method="init"></bean>
5.java.lang.ClassNotFoundException: org.springframework.web.servlet.DispatcherServlet解决：添加servlet-api.jar
6.org.springframework.web.servlet.DispatcherServlet cannot be cast to javax.servlet.Servlet解决：（与tomcat自带的servlet-api冲突） 
  1. <dependency>
  2.   <groupId>javax.servlet</groupId>
  3.     <artifactId>javax.servlet-api</artifactId>
  4.     <version>3.1.0</version>
  5. </dependency>
修改为（加上scope） 
  1. <dependency>
  2.   <groupId>javax.servlet</groupId>
  3.     <artifactId>javax.servlet-api</artifactId>
  4.     <version>3.1.0</version>
  5.     <scope>provided</scope>
  6. </dependency>
7.Could not resolve placeholder 'jdbc.driver' in string  value "${jdbc.driver}"
错误缘由：在配置文件中，使用两次引入配置文件的扫描，但是两次扫描的key是一样的，并不能进行两次扫描
解决方法一：将配置文件写法哦一个方法里，用“,”隔开 
<context:property-placeholder location="classpath:config/resource.properties" />
<context:property-placeholder location="classpath:config/db.properties" />
解决方法二：在每个扫描的配置文件后加上ignore-unresolvable="true" 
<context:property-placeholder location="classpath:config/resource.properties" ignore-unresolvable="true" />
<context:property-placeholder location="classpath:config/db.properties" gnore-unresolvable="true" />
